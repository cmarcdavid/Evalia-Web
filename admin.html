Connect to GitHub

Enter your GitHub Personal Access Token once — stored only in your browser.

Get token (2 min):
1. github.com/settings/tokens/new
2. Note: "Evalia Admin" · No expiration · Tick repo
3. Generate → copy → paste above

What the token does: The admin uses your GitHub token to save content (properties, experiences, settings, media) directly to your GitHub repo. Guests and bookings are stored in Supabase — those don't need the token.

⚠️ Rate limited? GitHub limits API calls. Wait a few minutes then try again — your token is fine.

Guest Profile
Integrations: QuickBooks · Zoho · Stripe · WhatsApp Business
💼 QuickBooks: Export CSV → Import as Customers. Fields: Name, Email, Phone, Nationality, Arrival, Departure.
📊 Zoho CRM: Use CSV export → Contacts import. Or sync _data/guests/ via Zoho's GitHub connector.
💳 Stripe: Guest emails ready for Stripe Customer creation. Use the API or paste email list into Stripe dashboard.
📱 WhatsApp Business: Phone numbers stored in international format. Ready for WhatsApp Business broadcast lists.
Reviews & Testimonials
Reviews submitted via the Write a Review button on the website appear here as Pending. Approve to publish them in the Testimonials section. Reject to hide. Stored in _data/reviews/ on GitHub and in the Supabase reviews table.
⚠️ If reviews submitted on the website don't appear here, run this SQL in Supabase: CREATE TABLE IF NOT EXISTS reviews (id UUID DEFAULT gen_random_uuid() PRIMARY KEY, name TEXT, text TEXT, service TEXT, email TEXT, rating INTEGER, status TEXT DEFAULT 'pending', date TIMESTAMPTZ DEFAULT NOW(), avatar TEXT); ALTER TABLE reviews ENABLE ROW LEVEL SECURITY; CREATE POLICY "public_insert_reviews" ON reviews FOR INSERT WITH CHECK (true); CREATE POLICY "public_read_approved" ON reviews FOR SELECT USING (true);
